Design Notes

  • Beveled frame borders — different color per side simulates directional lighting
  • Inset shadows on each flap cell create physical depth
  • Golden-amber text (#f0d8a0) with subtle glow mimics aged plastic lettering
  • Hinge line at 50% with micro-highlight edge
  • Ambient warm glow on the frame
  • Font stays Courier New to match hardware character set

Usage Guidelines

Architecture

  • tokens.css — Design tokens only (CSS custom properties in :root). No selectors.
  • components.css — All component classes. Uses tokens exclusively — no raw hex values.
  • utilities.css — Helper classes for text, spacing, layout, and effects.

Naming Convention

  • BEM-lite with sf- prefix
  • Block: .sf-btn, .sf-card
  • Element: .sf-card__title
  • Modifier: .sf-btn--primary, .sf-btn--sm
  • Variant on block: .sf-card--app

Do / Don't

  • ✓ Use token variables for all colors, spacing, and sizes
  • ✓ Use --ease-snap for interactive transitions
  • ✓ Keep border-radius at 0-4px (brutalist aesthetic)
  • ✓ Use eyebrow text for section labels
  • ✕ No raw hex values in component styles
  • ✕ No border-radius above 4px (except toggle switch pill)
  • ✕ No cool blues or pure whites — everything warm-shifted
  • ✕ Don't change the flap font (must match hardware)

Aesthetic Principles

  1. Mechanical warmth — Every surface should feel like part of a precision-engineered machine. Warm browns, not cold grays.
  2. Physical depth — Use inset shadows on interactive elements. Buttons should feel pressable. Flaps should feel recessed.
  3. Editorial hierarchy — Tracked uppercase eyebrow labels, display serif for headings, clean sans for body. The UI reads like a well-typeset departure board.
  4. Restrained accent — Orange accent used sparingly for active states and CTAs. The dominant palette is warm neutrals.
  5. Analog texture — Film grain overlay adds subtle analog warmth. Beveled frame borders suggest real materials.
